Reports FunC-style constant functions that should be converted to proper constants.

In Tolk, instead of using functions with asm blocks containing only PUSHINT operations, you should use const declarations for better readability and performance.

Example:

// Bad:
fun OP_INTERNAL_TRANSFER(): int
    asm "0x178d4519 PUSHINT";

// Good:
const OP_INTERNAL_TRANSFER = 0x178d4519;

The inspection provides quick fixes to:

  • Convert a single function to a constant
  • Convert all FunC-style constants in the file
  • Update all function calls to reference the new constants
