Reports expressions with potentially confusing operator precedence that may lead to unexpected behavior.

This inspection detects cases where mixing comparison operators (<, >, ==, etc.) with logical operators (|, &, ^) without explicit parentheses can create confusion about the intended evaluation order.

In FunC, logical operators have higher precedence than comparison operators, which can lead to unexpected results.


Problematic: 

cpu > max_cpu | cpu < min_cpu
// Parsed as: cpu > (max_cpu | cpu) < min_cpu
Better: 

(cpu > max_cpu) | (cpu < min_cpu)
// Clear intended meaning

The inspection helps prevent logical errors by encouraging explicit parentheses in mixed operator expressions.
