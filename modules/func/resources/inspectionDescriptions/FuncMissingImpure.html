Reports functions returning unit type () that may need the impure specifier to prevent compiler optimization.

In FunC, functions that return nothing (unit type ()) typically have side effects and should be marked with the impure specifier. Without the impure specifier, the compiler may optimize away such functions completely, leading to unexpected behavior and debugging difficulties.


Problematic (may be optimized away): 

() send_message(cell msg) {
    send_raw_message(msg, 0);
}
Correct (protected from optimization): 

() send_message(cell msg) impure {
    send_raw_message(msg, 0);
}

The inspection provides a quick fix to add the impure specifier automatically.
