Reports unused include statements in FunC code.

This inspection identifies include statements that do not contribute any symbols to the current file. It performs intelligent analysis considering:

  • Direct usage - symbols used directly in the current file
  • Transitive dependencies - symbols imported through chains of includes
  • Re-export scenarios - includes needed by files that import the current file

The inspection automatically excludes stdlib.fc and stdlib.func from analysis as they are commonly needed system libraries.

Examples of unused includes:


#include "math.fc"     // Contains: add(int, int), multiply(int, int)
#include "strings.fc"  // Contains: concat(slice, slice), length(slice)

() main() impure {
    // Only using functions from strings.fc
    slice result = concat("hello", "world");
    
    // math.fc is unused - will be flagged
}

Examples of includes that appear unused but are actually needed:


// utils.fc
#include "math.fc"    // Not used directly here, but needed for re-export

// main.fc  
#include "utils.fc"

// other.fc
#include "main.fc"
() other() {
    int sum = add(1, 2);  // Uses add() from math.fc via utils.fc -> main.fc
    // math.fc include in utils.fc is NOT flagged as unused
}

This inspection helps maintain clean, efficient code by removing unnecessary dependencies while preserving includes required for proper functionality.
